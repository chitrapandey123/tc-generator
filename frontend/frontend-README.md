# TC Generator — Frontend (React)

React frontend for the AI-powered test case generator tool.

---

## Tech Stack
- React 18
- Vite — build tool and dev server
- Axios — HTTP client for API calls

---

## Setup

```bash
# Install dependencies
npm install

# Run development server
npm run dev
# Runs at http://localhost:5174

# Build for production
npm run build
```

## Component Overview

### `App.jsx`
Main component that manages the 4-step flow and global state.

**State managed:**
- `step` — current step (0-3)
- `project` — Jira project key
- `stories` — fetched from Jira
- `testCases` — generated by Claude
- `pushResults` — results from Xray push
- `loading` — loading state
- `error` — error messages

**Flow:**
```
Step 0: Connect    → enter project key → fetch stories
Step 1: Stories    → select stories → generate TCs
Step 2: Review     → edit TCs → push to Xray
Step 3: Results    → see created TCs with links
```

---

### `api.js`
All API calls to the FastAPI backend.

```javascript
fetchStories(project)                          // GET /api/stories
generateTCs(stories)                           // POST /api/generate-tc
getXrayToken()                                 // POST /api/xray/token
pushToXray(projectKey, storyKey, tcs, token)   // POST /api/xray/push
```

**Base URL:** `http://localhost:8000`

---

### `StoriesList.jsx`
Displays fetched Jira stories with:
- Checkbox selection (single and select all)
- Story key, summary, status, priority, assignee
- Description preview
- Generate button (disabled until at least 1 selected)

---

### `ReviewTC.jsx`
Displays AI-generated test cases with:
- Checkbox to include/exclude each TC
- Inline editing of TC title
- Inline editing of each step (action, data, result)
- Delete individual TCs
- Color-coded TC type badges
- Push selected TCs button

---

### `PushResults.jsx`
Displays push results with:
- Summary cards (created, failed, total)
- Per-TC status with direct link to Jira issue
- Error message if TC failed
- Start over button

---

## Connecting to Backend

The frontend expects the FastAPI backend running at `http://localhost:8000`.

Make sure backend is running before using the UI:
```bash
cd ../jira-claude-backend
uvicorn main:app --reload
```

To change the backend URL, update `src/api.js`:
```javascript
const BASE_URL = "http://localhost:8000"; // change this
```
